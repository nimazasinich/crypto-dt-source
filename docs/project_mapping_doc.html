1. System Overview

The Crypto Intelligence Hub is a comprehensive platform integrating real Hugging Face crypto models with external data providers. The system provides sentiment analysis, market data, news aggregation, and trading signals through a unified API and web interface.

Architecture Components

Component File Purpose
AI Models ai_models.py Model registry and Hugging Face integration
API Server api_server_extended.py FastAPI backend with all endpoints
Main Entry hf_unified_server.py ASGI app entry point for HF Spaces
Frontend index.html, static/js/app.js Web UI with tabs for all features
Config config.py Model configurations and settings

2. AI Models

The system integrates 5 primary Hugging Face models for crypto analysis, plus fallback models.

Crypto sentiment binary classification model trained on cryptocurrency-related text. Provides sentiment analysis specifically tuned for crypto market language.

ðŸ”¹ ElKulako/cryptobert

Crypto social sentiment classifier returning Bullish/Neutral/Bearish labels. Optimized for social media and news content related to cryptocurrencies.

Financial tweet sentiment analysis model for market-related social media content. Specialized in analyzing financial tweets and news headlines.

Crypto and DeFi text generation model for analysis and content creation. Can generate insights, summaries, and analysis of crypto market conditions.

4. Frontend Integration

Tab Structure

Tab HTML ID Main Functions Backend Endpoints
Dashboard #tab-dashboard loadDashboard() /api/status, /api/resources
Market #tab-market loadMarketData() /api/market, /api/trending, /api/sentiment

Sentiment #tab-sentiment analyzeSentiment(), analyzeGlobalSentiment(), analyzeNewsSentiment() /api/sentiment/analyze, /api/models/list
News #tab-news loadNews() /api/news/latest
Providers #tab-providers loadProviders(), searchResources() /api/providers, /api/resources/search
Diagnostics #tab-diagnostics runDiagnostics(), loadDiagnostics() /api/diagnostics/run, /api/status, /api/logs/errors
API Explorer #tab-api-explorer testAPI() Any endpoint (user-selected)

Key JavaScript Functions

populateSentimentModelDropdown()

Called via loadSentimentModels() on page load. Fetches models from /api/models/list and populates the sentiment model dropdown.

analyzeSentiment()

Reads user input from #sentiment-text, #sentiment-mode, and #sentiment-model. Sends POST to /api/sentiment/analyze with optional model_key. Displays result in #sentiment-result with emoji, label, confidence.

loadProviders()

Fetches providers from /api/providers. Renders provider table in #providers-list with status badges and response times.

loadDashboard()

Loads system overview from /api/status and /api/resources. Updates stat cards and creates Chart.js category chart.

5. Deployment Guide

Environment Variables

Variable Values Description
HF_MODE off, public, auth Hugging Face model loading mode
HF_TOKEN string Hugging Face API token (for auth mode)
PORT integer Server port (default: 7860)
USE_FASTAPI_HTML true/false Use FastAPI+HTML mode instead of Gradio

Starting the Server

# Local development
uvicorn hf_unified_server:app --reload --port 7860

# Docker
docker-compose up

# Hugging Face Spaces
# Automatically uses hf_unified_server.py as entry point

Model Loading Modes

  • off: No models loaded, uses fallback lexical analysis
  • public: Loads public models without authentication (recommended)
  • auth: Requires HF_TOKEN for private/gated models

Testing Endpoints

Use the API Explorer tab in the web UI, or test directly with curl: 

# Test health
curl http://localhost:7860/api/health

# List models
curl http://localhost:7860/api/models/list

# Analyze sentiment
curl -X POST http://localhost:7860/api/sentiment/analyze \
  -H "Content-Type: application/json" \
  -d '{"text": "Bitcoin is bullish!", "mode": "crypto"}'
